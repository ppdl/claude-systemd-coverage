# 프로젝트 작업 지침

## 작업 기준 문서

- **`docs/coverage-agent-runbook.md`**: 에이전트가 처음부터 끝까지 단독으로 실행하기 위한 기준 문서.
  작업 수행 시 이 파일을 참조한다.
- **`docs/coverage-measurement-guide.md`**: 작업 과정과 배경지식을 기록한 참고 문서.
  기준 문서에 없는 배경 정보가 필요할 때 참조한다.

## 문서화 규칙

### 에러 발생 시 — troubleshooting.md 먼저 확인

작업 중 에러가 발생하거나 의도하지 않은 동작이 발생하면 **반드시 아래 순서를 따른다**:

1. **`docs/troubleshooting.md`를 먼저 확인**한다.
   동일하거나 유사한 에러가 기록되어 있는지 검색한다.
2. 기록된 해결책이 있으면 그것을 적용한다.
3. 새로운 에러이거나 기존 해결책으로 해결되지 않으면 직접 원인을 파악하여 해결한다.
4. 해결 후 **`docs/troubleshooting.md`에 즉시 기록**한다:
   - 에러 증상
   - 원인
   - 시도한 방법 (실패한 것 포함)
   - 최종 해결책
5. 이슈 해결 후 `docs/coverage-agent-runbook.md`의 업데이트가 필요한지 검토한다.
   업데이트가 필요하다고 판단되면 **사용자에게 업데이트 수행 여부를 먼저 묻고**, 허락받은 경우에만 진행한다.

> **중요:** 에러 해결 내용을 `docs/coverage-agent-runbook.md`에 곧바로 반영하지 않는다.
> runbook은 최종 작동하는 절차만 담고, 에러 기록은 troubleshooting.md에서 관리한다.

### 작업 진행 시 — runbook 업데이트 타이밍

`docs/coverage-agent-runbook.md`는 아래 경우에만 업데이트한다:
- 단계가 완료되어 runbook의 절차 자체가 변경되어야 할 때
- 사용자의 명시적 허락을 받은 경우

`docs/coverage-measurement-guide.md`는 배경 정보 참고용이며, 작업 중 자동으로 업데이트하지 않는다.

## 작업 방식

- 코드 탐색이나 에이전트 실행 전에 반드시 사용자와 계획을 먼저 논의한다.
- sdb 관련 작업은 `/sdb` 커스텀 슬래시 커맨드를 참조한다.
